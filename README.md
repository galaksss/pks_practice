<p>В рамках задания было спроектировано мини-приложение <strong>QuickTasks</strong> для ведения простого списка задач.</p>

<p>Были созданы три экрана:</p>
<ul>
  <li><strong>Welcome</strong> — стартовый экран приветствия с названием приложения и кнопкой перехода.</li>
  <li><strong>Main</strong> — главный экран со списком задач и плавающей кнопкой добавления новой задачи.</li>
  <li><strong>Add Item</strong> — экран добавления задачи с полем ввода и кнопками «Сохранить» и «Назад».</li>
</ul>

<p>Для построения интерфейса были использованы базовые компоненты Figma:</p>
<ul>
  <li><strong>Фреймы (Frame) размера 412×917</strong> — в роли экранов мобильного приложения.</li>
  <li><strong>Текстовые элементы (Text)</strong> — для названия приложения, подзаголовков («Простое приложение для заметок», «Список задач», «Новая задача») и подписей кнопок.</li>
  <li><strong>Прямоугольники (Rectangle)</strong> — в качестве кнопок («Начать», «Сохранить»), карточек задач на главном экране и поля ввода на экране добавления задачи.</li>
  <li><strong>Эллипс (Ellipse)</strong> — круглая плавающая кнопка с иконкой «+» для добавления новой задачи.</li>
  <li><strong>Группы</strong> — для объединения кнопок с текстом и карточек задач.</li>
  <li><strong>Layout grid</strong> — сетка в фреймах для выравнивания блоков и сохранения одинаковых отступов.</li>
</ul>

<h3>Организация навигации (режим Prototype)</h3>

<p>Навигация настраивалась в правой панели Figma в режиме Prototype. Для интерактивного прототипа были созданы следующие связи:</p>
<ul>
  <li>На экране <strong>Welcome</strong> кнопка «Начать»:
    <br>On tap → Navigate to → <strong>Main</strong> — переход на главный экран.
  </li>
  <li>На экране <strong>Main</strong> плавающая кнопка «+»:
    <br>On tap → Navigate to → <strong>Add Item</strong> — переход на экран добавления задачи.
  </li>
  <li>На экране <strong>Add Item</strong>:
    <ul>
      <li>кнопка «Сохранить»:
        <br>On tap → Navigate to → <strong>Main</strong> — возвращение на главный экран;
      </li>
      <li>кнопка «← Назад»:
        <br>On tap → Navigate to → <strong>Main</strong> — возврат без добавления новой задачи.
      </li>
    </ul>
  </li>
</ul>

<h3>Краткий анализ</h3>

<h4>Что получилось удобно</h4>
<ul>
  <li>Интерфейс получился минималистичным и понятным: на каждом экране всего несколько элементов, поэтому пользователю легко ориентироваться.</li>
  <li>Чётко разделены роли экранов: приветствие, список задач и создание задачи.</li>
  <li>Крупные кнопки и читаемые шрифты хорошо смотрятся на экране 412×917 и подходят для управления пальцем.</li>
  <li>Навигация линейная и однозначная: с приветствия — к списку, от списка — к добавлению, от добавления — назад.</li>
</ul>

<h4>Что можно улучшить</h4>
<ul>
  <li>Добавить визуальную обратную связь (например, изменение цвета кнопки при нажатии).</li>
  <li>Реализовать состояние списка после добавления задачи (отображение новой карточки на главном экране).</li>
  <li>Продумать дополнительные экраны: редактирование и удаление задач, настройки приложения, фильтрацию по статусу.</li>
</ul>
